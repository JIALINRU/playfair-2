Overview

The Playfair cipher is a pencil and paper cipher designed in 1854 by Charles Wheatstone but named after Lord Playfair who promoted its use.

The Key

The key is a 5x5 Polybius square containing 25 of the 26 capital letters of the English alphabet. In this application, I and J are combined. In others, letters such as Q or X may be dropped. The order of the letters within the square is critical. If the person with whom you're exhanging the message doesn't have their square in the same order as you, then they will not be able to correctly exchange messages with you. The sequencing of the key is made more simple by using a Keyphrase.

The Keyphrase

The keyphrase is an alphabetic sequence (usually a word or set of words) which are used to change the order of the Polybius Square. You and the person with whom you are exchanging the message must have the same key to decipher the other's messages. Because the most sensitive part of the process is the key, you should keep the keyphrase secret and ensure a mechanism of rotating the keyphrase. A common method of rotating keys is to use a book which both of you have and a pre-arranged algorithm for finding the day's keyphrase within that book.

If you wish to specify the full square rather than just a beginning keyphrase, simply enter all 25 letters of the key into the Keyphrase box

Encrypting

  • Enter the keyphrase into the Keyphrase box
  • Enter the message you wish to encrypt into the Message box
  • Click the Encrypt button
  • The encrypted message appears in the Message box

Remember that the alphabet for this is 25 upper-case letters. So everything that is not a letter is removed, and all letters are converted to upper-case. If you check the Convert "." to "STOP" checkbox, all periods will be changed to the word "STOP" so that you at least get sentence endings in the result. Similarly, if you check Spell out digits, numbers will be spelled out digit-by-digit.

Decrypting

  • Enter the keyphrase into the Keyphrase box
  • Enter the encrypted message into the Message box
  • Click the Decrypt button
  • The plaintext message appears in the Message box

"The decrypted message isn't the same as I put it in." or "I can't read the new decrypted message"

Remember, this is an alphabet of only 25 characters, and doesn't include punctuation. So the resulting plaintext will probably not be exactly the same as you originally entered. Here are some hints to get you started:
  • You'll need to figure out where word boundaries are. When constructing your plaintext message, help out the recipient by removing any ambiguity in the boundaries of the plaintext message. For example: "TOPANDASYNCHRONOUS" might be construed as "TOP AND A SYNCHRONOUS" or "TOP AND ASYNCHRONOUS" or even "TO PANDA SYNCRHONOUS"
  • J's from the plaintext are converted to I's.
  • To avoid letter frequency issues of normal substitution ciphers, the Playfair cipher converts a message into pairs of letters. A single pair cannot contain a repeating letter, so an "X" will be inserted between them. If the decrypted message contains "NOXON", the original was probably "NOON.
  • Even if Convert "." to "STOP" and Spell out digits are selected, these processes will not be reversed when decrypting. There's simply no good automated way to determine if "PASTOPEN" means "past open" or "pa. en" or if "FALSEVENT" should be "FAL7VENT" or "FALSE VENT".
  • If the last digraph only includes one letter, a "Z" will be appended. If the last "Z" doesn't make sense, just drop it.
